Color Space Comparison

CSS Color Level 4 introduces new color spaces beyond sRGB. Each space has different gamuts and perceptual properties.

sRGB vs Display P3 vs Rec.2020 gamut comparison

sRGB

Standard web colors. Covers ~35% of human vision. All browsers, all displays.

rgb(255,0,0)
rgb(0,128,0)
rgb(0,0,255)

Display P3

Wider gamut. ~25% more colors than sRGB. Most modern displays. Apple standard.

color(display-p3 1 0 0)
color(display-p3 0 0.5 0)
color(display-p3 0 0 1)

Rec.2020

Ultra-wide gamut. ~75% of visible spectrum. Few displays. Future-proof.

color(rec2020 1 0 0)
color(rec2020 0 0.5 0)
color(rec2020 0 0 1)

OKLCH Color Picker

OKLCH is perceptually uniform: equal steps in lightness, chroma, or hue produce equal visual differences. This makes it ideal for generating harmonious palettes.

Picker oklch(L C H)

How gamut mapping works: When you specify a color outside the display's gamut (e.g., oklch(0.8 0.4 150) on an sRGB display), the browser reduces the chroma (saturation) until the color fits within the displayable range. This preserves the perceived hue and lightness as closely as possible.

Wide-Gamut Display Detection

Detect if the user's display supports wide-gamut colors using @media (color-gamut: p3) and JavaScript. Serve appropriate colors based on display capability.

Your Display detection

/* CSS: Progressive color enhancement */ .accent { /* sRGB fallback */ color: rgb(59, 130, 246); } @media (color-gamut: p3) { .accent { /* Wider gamut for P3 displays */ color: oklch(0.64 0.2 250); } } @media (color-gamut: rec2020) { .accent { /* Ultra-wide gamut */ color: color(rec2020 0.2 0.4 0.9); } } /* JavaScript detection */ const supportsP3 = matchMedia('(color-gamut: p3)').matches; const supportsRec2020 = matchMedia('(color-gamut: rec2020)').matches; const colorDepth = screen.colorDepth; // 24 = 8-bit, 30 = 10-bit
